### 1.3 右侧面板（输出预览区）

#### 功能描述
- **分割结果显示**: 实时显示SAM3分割结果
- **原图 + 掩码融合**: 使用屏幕混合模式叠加
- **多颜色编码**: 不同分割区域用不同颜色表示，颜色和prompt分组颜色一致

#### 颜色方案
```
分割输出颜色（HSV循环）:
- 区域1: 紫红色 (Magenta)
- 区域2: 绿色 (Green)
- 区域3: 黄色 (Yellow)
- 区域4: 蓝色 (Blue)
- 区域5-10: 循环使用不同色调
- 透明度: 50-70% (半透明显示原图)

分割区域边界: 清晰的轮廓线
- 线宽: 2-3px
- 颜色: 与区域颜色相同
```

#### 交互功能
```
┌─ 分割统计 ──────────────┐
│ 区域数: 5               │
│ 最大区域: 12.5% 面积    │
│ 平均置信度: 0.95        │
├─────────────────────────┤
│ [导出掩码]              │ ← PNG格式
│ [导出分割]              │ ← JSON格式  
│ [复制到剪切板]          │ ← 用于其他使用
└─────────────────────────┘
```


## 4. 用户交互场景

1. 上传图像
2. 添加1-6个Prompt组，选中每个组，依次添加正向点 (想要的区域)和负向点 (排除不想要的区域)
3. 点击 【开始切割】，调用SAM开始计算，SAM根据图像和提示点，对画面进行分割，并将分割结果用同色线框标记出来
4. 观察右侧分割效果
5. 迭代调整 (增加/删除提示)
6. 完成后导出结果

## 6. 导出功能

### 6.1 导出提示 (Export Points)

**格式**: JSON
```json
{
  "export_time": "2026-04-12T10:30:00Z",
  "image_info": {
    "filename": "carA.webp",
    "width": 1920,
    "height": 1440,
    "size_kb": 256
  },
  "prompts": {
    "positive_points": [[120, 150], [450, 280]],
    "negative_points": [[300, 200]],
    "positive_boxes": [[100, 80, 200, 180]],
    "negative_boxes": []
  }
}
```

### 6.2 导出掩码 (Export Mask)

**格式**: PNG (8-bit灰度)
- 每个像素值代表区域ID
- 0 = 背景
- 1,2,3,... = 不同分割区域


## 7. 错误处理与用户反馈

### 7.1 错误场景

| 错误类型 | 原因 | 用户反馈 |
|--------|------|--------|
| 未加载图像 | 点击分割前未上传 | ⚠️ "请先上传图像" (红色) |
| 无有效提示 | 未标记任何点/框 | ⚠️ "请标记至少一个提示点" |
| 模型加载失败 | GPU内存不足 | ❌ "模型加载失败：CUDA OOM" |
| 推理超时 | 后端卡住 | ⏱️ "处理超时(>30s)，请重试" |
| 网络错误 | 前后端通信失败 | 🔄 "网络连接失败，重试..." |

### 7.2 成功反馈

```
  "处理中 (2.1/5.0秒)"

完成提示:
  ✓ 分割完成 (2.1秒)
  5个区域已识别
  [导出] [继续编辑]
```
